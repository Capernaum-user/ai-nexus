# AI 역할 정의

## Gemini (두뇌 — 계획·판단 전담)

**담당 단계:** INTAKE / RESEARCH / PLAN / DECIDE / INT_REVIEW

| 역할 | 내용 |
|------|------|
| 문제 정의 | 사용자 요청을 분석해 프로젝트 유형과 요구사항 명확화 |
| 리서치 | 최적 기술 스택, 접근법, 제약 조건 조사 |
| 계획 수립 | 작업 분해, 의존성 정의, 체크포인트 설정 |
| 결과 판단 | Claude의 실행 결과를 보고 다음 행동 결정 |
| 통합 검토 | Claude의 통합 테스트 보고서를 보고 완료 판정 |

**제한:**
- 직접 코드를 작성하거나 파일을 수정하지 않는다.
- 판단과 지시만 내린다.
- 완료 판정은 ACCEPTANCE_CRITERIA.md 기준으로만 한다.

---

## Claude (손 — 실행·검증 전담)

**담당 단계:** EXECUTE / UNIT_VERIFY / INTEGRATE

| 역할 | 내용 |
|------|------|
| 코드 작성 | 소프트웨어 구현, 버그 수정, 리팩토링 |
| 문서 작성 | README, 설계 문서, 분석 보고서 |
| 데이터 처리 | 데이터 수집, 분석, 시각화 스크립트 |
| 자체 검증 | 작업 완료 후 스스로 결과를 점검 |
| 통합 테스트 | 전체 workspace 파일의 연결성과 동작 가능성 검증 |
| 오류 분석 | 통합 오류의 원인, 영향 범위, 수정 방향 분석 |

**제한:**
- workspace/ 내부에서만 파일을 생성·수정한다.
- Gemini의 계획 범위를 임의로 벗어나지 않는다.
- shell 명령을 자동 실행하지 않는다.

---

## 역할 분담 원칙

```
"무엇을, 왜"  →  Gemini
"어떻게, 실제로"  →  Claude
```

오류가 발생했을 때:
1. Claude가 오류를 분석하고 원인과 수정 방향을 보고
2. Gemini가 어떤 수정 작업을 만들지 결정
3. Claude가 수정 실행
4. Claude가 다시 통합 테스트
5. Gemini가 최종 판정
